# IBL Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 为 PBR 渲染管线添加运行时 IBL（漫反射 irradiance + 镜面反射 pre-filtered env + BRDF LUT）

**Architecture:** 新增 IBLGenerator 类，从全景 HDR 天空盒纹理运行时生成 cubemap 和预计算贴图。扩展 perFrameLayout 绑定 IBL 纹理，修改 Lit.slang 替换 ambient 为 IBL 采样。

**Tech Stack:** C++17, Vulkan 1.4, Slang shaders

**Spec:** `docs/design/2026-03-25-ibl-design.md`

## Task 1: IBL 着色器（4 个）

创建 4 个 IBL 生成着色器。所有着色器用全屏三角形顶点着色器 + cubemap face 方向计算。

### EquirectToCubemap.slang
- 输入: push constant (face index) + 全景 HDR sampler (binding 0)
- 输出: cubemap face color
- 逻辑: UV → cubemap direction → equirectangular UV → sample panorama

### IrradianceConvolution.slang
- 输入: push constant (face index) + HDR cubemap (binding 0)
- 输出: irradiance color
- 逻辑: 对法线方向的半球做 cosine-weighted 积分，~64 采样

### PrefilterEnvMap.slang
- 输入: push constant (face index, roughness) + HDR cubemap (binding 0)
- 输出: pre-filtered color
- 逻辑: GGX importance sampling，采样数 = 128

### BrdfLut.slang
- 输入: 无纹理，仅 UV
- 输出: R16G16 (scale, bias)
- 逻辑: Hammersley 序列 + GGX importance sampling 积分

编译着色器并提交。

## Task 2: IBLGenerator 类骨架

创建 `IBLGenerator.h/cpp` 包含：
- `init()` / `destroy()` 生命周期
- `generate(panoramaView, sampler)` 主入口
- `generateBrdfLut()` 一次性生成
- 4 种资源的 Vulkan 对象（image/memory/view/sampler）
- 4 个 render pass（可复用相同格式的 pass）
- 4 条管线
- cubemap framebuffer 数组（6 面 × 多 mip）
- getter: `getIrradianceView()`, `getPrefilteredView()`, `getBrdfLutView()`, `getCubemapSampler()`

关键实现细节：
- Cubemap 用 `VkImage` with `VK_IMAGE_CREATE_CUBE_COMPATIBLE_BIT` + `arrayLayers=6`
- 每个 face 用 `VkImageView` with `baseArrayLayer=face, layerCount=1` 作为 framebuffer attachment
- Pre-filtered cubemap 有 5 mip levels
- BRDF LUT 格式 `R16G16_SFLOAT`，需要单独的 render pass

## Task 3: IBLGenerator 生成管线实现

实现 4 个生成 pass：

### generateCubemap(): 6 面渲染
- 创建临时 framebuffer（每面一个）
- 循环 6 面：begin render pass → bind pipeline → push constant (faceIndex) → draw → end
- 渲染完成后转换 cubemap layout 到 SHADER_READ_ONLY

### generateIrradiance(): 6 面卷积
- 同上模式，采样源为 cubemap

### generatePrefiltered(): 6面 × 5mip
- 双重循环：mip (roughness) × face
- 每个 mip 的 framebuffer 尺寸递减（128→64→32→16→8）
- push constant 传 roughness = mip / 4.0

### generateBrdfLut(): 单次全屏 pass
- 512×512 render target，R16G16_SFLOAT 格式

## Task 4: Renderer 集成

### 扩展 perFrameLayout
当前 2 binding（UBO + shadow），扩展为 5 binding：
- binding 0: UBO (不变)
- binding 1: Shadow map (不变)
- binding 2: Irradiance cubemap
- binding 3: Pre-filtered cubemap
- binding 4: BRDF LUT

### 修改 descriptor set 创建和更新
- `createPerFrameSets` 后写入 IBL 纹理到 binding 2/3/4
- IBL 未生成时绑定黑色 fallback（1x1 cubemap）
- IBL 生成后更新 descriptor set（带 vkDeviceWaitIdle 保护）

### 触发 IBL 生成
- `createOffscreen()` 末尾（天空盒已加载后）调用 `m_iblGenerator.generate()`
- 需要在天空盒纹理可用后调用

### 所有使用 perFrameLayout 的 pipeline 需要重建
因为 layout 从 2 binding 变为 5 binding，所有引用它的 pipeline 都需要用新 layout 重建。

## Task 5: Lit.slang IBL 采样

替换 Lit.slang 中的环境光计算：

```hlsl
// 旧代码
float3 ambient = frame.ambientColor * albedo;

// 新代码
float3 N = normal;
float3 V = normalize(frame.cameraPos - input.worldPos);
float3 R = reflect(-V, N);
float NdotV = max(dot(N, V), 0.0);

// 漫反射 IBL
float3 irradiance = irradianceMap.Sample(N).rgb;
float3 kS = fresnelSchlick(NdotV, F0);
float3 kD = (1.0 - kS) * (1.0 - metallic);
float3 diffuseIBL = irradiance * albedo * kD;

// 镜面反射 IBL
float3 prefilteredColor = prefilteredMap.SampleLevel(R, roughness * 4.0).rgb;
float2 brdf = brdfLUT.Sample(float2(NdotV, roughness)).rg;
float3 specularIBL = prefilteredColor * (F0 * brdf.x + brdf.y);

float3 ambient = diffuseIBL + specularIBL;
```

新增 3 个 sampler 绑定：
```hlsl
[[vk::binding(2, 0)]] SamplerCube irradianceMap;
[[vk::binding(3, 0)]] SamplerCube prefilteredMap;
[[vk::binding(4, 0)]] Sampler2D brdfLUT;
```
